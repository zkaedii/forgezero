## Known issues at release

- HYGIENE-FORGE-005: paths_touched evaluator occasionally reported 'in/forge0.ts' alongside 'bin/forge0.ts' in --explain output. Assumed transient artifact; not reproducible in clean state.
- HYGIENE-FORGE-006: kickoff ledger summary truncates session UUID to first 8-char segment. Cross-event correlation in ledger list requires manual UUID expansion. Suggested fix: include full session_id in summary, or stop truncating in title.
- HYGIENE-FORGE-007: two pre-existing 'any' casts at bin/forge0.ts ~lines 703 and 851 (opts.mode as any). Predates v0.2.0 work.
- HYGIENE-FORGE-008: forge0 verify --mode release --remote compares the remote tag-object SHA against the local commit SHA, instead of dereferencing annotated tags first via the peeled ref (refs/tags/X^{}). Causes verify to spuriously report yellow for any annotated release tag. v0.2.0 is the first annotated release; the bug surfaced naturally. One-line fix in src/verify/verify.ts remote tag check.
- HYGIENE-FORGE-009: forge0 ledger record CLI only exposes 'verify' and 'receipt' event kinds despite 'manual' being defined in LedgerEventKind. No CLI surface for free-text forensic notes.
- HYGIENE-FORGE-010: verify event honesty bound says 'does not prove remote CI completion' even when --ci was used and CI was successfully observed. Bound text should be conditional on whether --ci was applied.
